# ナレッジマネジメントシステム 説明サイト

## 概要

このサイトは、ナレッジマネジメントシステムの使い方や使う理由を説明するためのWebサイトです。

### 最終ゴール
**AIチャットボットで問い合わせゼロへ - 自己解決型の組織を実現**

## サイト構成

```
knowledge-management-system/
│
├── index.html                          # トップページ
│
├── guides/                             # 導入ガイド(段階的な理解)
│   ├── index.html                      # ガイド目次
│   ├── quickstart.html                 # クイックスタート(5分)
│   ├── vision.html                     # ビジョン説明(10分)
│   ├── four-categories.html            # 文書体系を4つに分ける理由(5分)
│   ├── knowledge-vs-manual.html        # ナレッジとマニュアルの違い(5分)
│   └── faq.html                        # よくある質問
│
└── assets/                             # 共通リソース(今後追加予定)
    ├── css/
    ├── js/
    └── images/
```

## 推奨される読み方

### 初めての方(5分)
1. **クイックスタート**を読む
   - 今日から使い始めるための最小限の知識

### 基本を理解したい方(25分)
1. **クイックスタート**(5分)
2. **ビジョン説明**(10分)
3. **文書体系を4つに分ける理由**(5分)
4. **ナレッジとマニュアルの違い**(5分)

### 運用中の方
- **よくある質問**で疑問を解消
- ハンドブック(今後追加予定)で詳細を参照

## 主な特徴

### 1. 段階的な理解を支援
- 5分で始められるクイックスタート
- 必要に応じて深く学べる構造

### 2. 2つの異なるニーズに対応
- **初めての人**: 導入ガイドで理解を助ける
- **運用中の人**: ハンドブック(今後)で詳細を参照

### 3. 最終ゴールの明確化
- AIチャットボット実現という目的を前面に
- そのために必要な設計であることを説明

### 4. 初見の人でも理解できる表現
- 専門用語を避ける
- 何を指しているのか明確にする

## よくある疑問への回答

### Q1: なぜ文書を4つに分けるのか?
**A**: AIチャットボット実現のためです。
- テキスト形式(①②③)でAI学習を可能に
- 人間用の図解資料(④)も提供
- 段階的なAI導入が可能

### Q2: ナレッジとマニュアルの違いは?
**A**: 目的と完成度が全く違います。
- **①ナレッジ**: 調査・試行錯誤の記録(60%でOK)
- **②マニュアル**: 確定した標準手順(100%必須)

### Q3: いつAIチャットボットができるのか?
**A**: 段階的に実現します。
- フェーズ1(今): Notionで基盤整備
- フェーズ2: システムG向けAIテスト
- フェーズ3: 全社公開準備
- フェーズ4: 全社AI展開

## GitHub Pagesへのデプロイ

1. GitHubリポジトリに`knowledge-management-system/`ディレクトリをプッシュ
2. Settings > Pages > Source で `main` ブランチと `/` (root) を選択
3. または、`/knowledge-management-system` フォルダを選択
